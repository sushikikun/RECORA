# Keiba Model Strategy Notes

競馬AIでは、目的変数によって正しいモデル方針、特徴量、評価指標、ROIへのつなぎ方が変わる。「とりあえずLightGBM」ではなく、何を予測したいかを先に決める。

## A. 目的別モデル戦略の基本方針

必ず以下を分ける。

- 3着以内予測
- 実際の複勝的中条件予測
- 1着予測
- 勝率予測
- 複勝率予測
- レース内順位予測
- 期待値判定
- 買い目評価

共通ルール:

- 的中率だけで判断しない。
- ベースラインと比較する。
- 時系列 train / valid / test で確認する。
- 確率を出すなら calibration を見る。
- 馬券評価では ROI、回収率、最大ドローダウン、ベット数を見る。
- ランキング指標とROIを混同しない。
- 少数サンプルの偶然勝ちを実力と判断しない。
- データリークを常に疑う。

## B. 3着以内予測 is_top3

目的:

- 初期モデル向き。
- 馬券圏内に来る馬を広く拾う。
- 最初の分類問題として使いやすい。

向いている場面:

- 初期ベースライン作成。
- 特徴量の効き方確認。
- 馬券圏内候補の抽出。
- 複勝率予測の前段階。

注意:

- 実際の複勝的中条件とは常に一致しない。
- 少頭数レースでは複勝条件とズレる。
- 3着以内に来る確率と、買って儲かるかは別。

候補モデル:

- ロジスティック回帰
- LightGBM
- CatBoost
- XGBoost

評価指標:

- accuracy
- precision
- recall
- AUC
- logloss
- Brier score
- calibration
- top3的中率
- ROI評価は別途行う

よくある失敗:

- 3着以内精度が高いだけで複勝ROIが高いと勘違いする。
- 人気馬ばかり拾って回収率が低くなる。
- 少頭数レースの扱いを間違える。

## C. 実際の複勝的中条件予測 is_place_hit_by_rule

目的:

- 複勝馬券の評価に合わせる。
- 出走頭数に応じた的中条件を使う。

方針:

- 8頭以上: 3着以内。
- 5〜7頭: 2着以内。
- 4頭以下: 複勝発売なし、または評価対象外。

向いている場面:

- 複勝ROI評価。
- 複勝回収率シミュレーション。
- 複勝ベースライン比較。

注意:

- `is_top3` と混同しない。
- `payout` と label を分ける。
- 複勝オッズが必要。
- 複勝発売条件を確認する。

評価指標:

- 的中率
- 回収率
- ROI
- 最大ドローダウン
- ベット数
- Brier score
- calibration

よくある失敗:

- 3着以内ラベルで複勝ROIを評価する。
- 少頭数レースで過大評価する。
- 複勝オッズの取得時点を無視する。

## D. 1着予測 / 勝率予測

目的:

- 単勝向け。
- 各馬の勝つ確率を推定する。
- オッズと比較して期待値を見る。

向いている場面:

- 単勝期待値評価。
- 勝率と単勝オッズのズレを見る。
- Benter的な「独自予測と市場評価のズレ」を検証する。

候補モデル:

- ロジスティック回帰
- LightGBM
- CatBoost
- XGBoost

評価指標:

- logloss
- Brier score
- calibration
- top1 accuracy
- 単勝ROI
- 最大ドローダウン
- オッズ帯別ROI

注意:

- 勝率予測では確率校正がかなり重要。
- 的中率は低くなりやすい。
- 高オッズ馬の勝率を過大評価しやすい。
- Favorite-Longshot Bias を疑う。
- 予測勝率 × オッズ が高くても、即買いとは判断しない。

よくある失敗:

- 的中率が低いからダメと早合点する。
- 期待値だけで買い判定する。
- calibrationを見ずにオッズ比較する。
- 最終オッズをリーク的に使う。

## E. 複勝率予測

目的:

- 馬券圏内に来る確率を予測する。
- 複勝・ワイド・馬券圏内評価の基礎にする。

向いている場面:

- 安定した確率予測。
- 人気馬の信頼度評価。
- 穴馬の馬券圏内可能性評価。
- 複勝ROIの検証。

候補モデル:

- ロジスティック回帰
- LightGBM
- CatBoost
- XGBoost

評価指標:

- logloss
- Brier score
- calibration curve
- 複勝的中率
- 複勝ROI
- 最大ドローダウン

注意:

- `is_top3` と `is_place_hit_by_rule` を分ける。
- 予測確率が高い馬ほど低配当になりやすい。
- 回収率はオッズとの関係で決まる。
- 人気帯別・オッズ帯別に性能を見る。

よくある失敗:

- 馬券圏内精度だけで優秀と判断する。
- 低配当ばかり拾ってROIが悪化する。
- 人気馬の確率を過大評価する。

## F. レース内順位予測 / Learning to Rank

目的:

- レース内で馬を相対的に並べる。
- 三連複・三連単に近い問題設定にする。
- 上位候補の順序を評価する。

向いている場面:

- 上位3頭候補の順位づけ。
- 三連系の前段階。
- 二値分類ではなく相対順位を見たい場合。

候補モデル:

- LightGBM Ranker
- CatBoost Ranker
- XGBoost ranking
- LambdaRank系

評価指標:

- top1 accuracy
- top3 accuracy
- NDCG
- MAP
- MRR

注意:

- NDCGやMAPが高くてもROIが高いとは限らない。
- ランキング精度と馬券評価は別。
- 三連単の買い目最適化とは別問題。
- ROI、回収率、最大ドローダウンで最終確認する。

よくある失敗:

- NDCGが高いから儲かると判断する。
- 順位予測だけで買い目を作る。
- オッズを無視して順位だけで買う。
- レース単位のgroup設計を間違える。

## G. 期待値判定モデル

目的:

- 予測確率とオッズを比較して、買う価値がある候補を探す。

基本:

- 期待値 = 予測確率 × オッズ

ただし、期待値が1.0を超えても即買いとは判断しない。

必要な確認:

- 予測確率が校正されているか。
- オッズ取得時点が正しいか。
- そのオッズで実際に買えるか。
- 控除率を考慮しているか。
- ベット数が少なすぎないか。
- 月別・競馬場別・オッズ帯別に安定しているか。
- 最大ドローダウンが許容範囲か。

評価指標:

- ROI
- 回収率
- 最大ドローダウン
- ベット数
- 平均払戻
- 月別収支
- オッズ帯別ROI
- 人気帯別ROI

よくある失敗:

- 期待値だけで買いと断定する。
- calibrationを無視する。
- サンプル数の少ない高ROIを信じる。
- 高オッズの偶然当たりで過大評価する。

## H. 買い目評価

目的:

- モデル予測を馬券評価に変換する。
- ただし、自動購入ではなく評価・検証までに留める。

対象:

- 単勝
- 複勝
- ワイド
- 馬連
- 三連複
- 三連単

初期方針:

- 最初は単勝・複勝・3着以内評価から始める。
- 三連複・三連単は後回し。
- 買い目点数を増やしすぎない。
- 券種ごとに目的変数と評価指標を分ける。

評価:

- 的中率
- 回収率
- ROI
- 最大ドローダウン
- ベット数
- 点数あたり回収率
- 月別安定性
- 条件別安定性

注意:

- 買い目最適化は過学習しやすい。
- 高配当の一撃で過大評価しない。
- 三連系は点数が増えやすい。
- 実運用や自動購入には踏み込まない。

## I. モデル開発の段階

v0:

- データ監査
- リーク確認
- 目的変数定義
- ベースライン作成

v1:

- 3着以内分類
- 基本特徴量のみ
- 時系列split
- ベースライン比較

v2:

- 過去成績特徴量
- 騎手・厩舎集計
- 集計母数とスムージング
- calibration確認

v3:

- オッズ・市場特徴量
- 期待値評価
- オッズ帯別・人気帯別ROI
- 最大ドローダウン

v4:

- 順位予測 / LTR
- 三連系の前段階評価
- NDCG / MAP / MRR
- ただしROI評価と分ける

v5:

- 条件別モデル
- 南関東4場別
- 距離別
- 馬場別
- 買い条件改善

注意:

- v0/v1を飛ばして複雑化しない。
- 強いモデルが出たらリークを再確認する。
- 改善は一度に一つずつ行う。

## J. 優秀なモデルの判定

優秀なモデルと呼ぶには、最低限以下を満たす。

- ベースラインに勝っている。
- trainだけでなくvalid/testでも崩れていない。
- 時系列テストで性能が保たれている。
- 回収率・ROIが複数期間で安定している。
- 最大ドローダウンが許容範囲。
- ベット数が少なすぎない。
- calibrationが破綻していない。
- オッズ帯別・人気帯別に偏りを確認済み。
- 南関東なら競馬場別に確認済み。
- データリークがない。
- 少数サンプルの偶然勝ちではない。
- 改善理由を説明できる。

## K. 危険なモデル改善

以下は危険な改善として警告する。

- 特徴量を増やせば良くなる。
- LightGBMなら勝てる。
- CatBoostならカテゴリが全部解決する。
- LTRなら三連単に強くなる。
- NDCGが高いから儲かる。
- 的中率が高いから優秀。
- 期待値が1.0を超えるから買い。
- 高配当が当たったからモデルが強い。
- 最終オッズを使えば正しい。
- 人気薄だけ狙えば妙味がある。
- 集計勝率をそのまま信じる。
- target encodingを雑に使う。
