## A. William Benter「Computer Based Horse Race Handicapping and Wagering Systems」

1. 資料名: William Benter「Computer Based Horse Race Handicapping and Wagering Systems」
2. 種類: 論文・実務報告
3. 信頼度: 高
4. 重要な教訓:
   - 競馬AIでは独自の馬評価だけでなく、大衆オッズが示す市場評価と比較する。
   - オッズは答えではなく、市場の集合知として扱う。
   - 予測モデル、オッズ、期待値、資金管理、検証を分けて考える。
   - 競馬AIは「当たりそうな馬」ではなく「市場評価よりズレている馬」を探す。
5. このスキルに入れる判断ルール:
   - 独自予測確率とオッズから見た市場評価を必ず比較する。
   - モデル評価では、予測精度、期待値、回収率、資金曲線を分けて確認する。
   - オッズを使う場合は、市場情報を借りていることを明示し、オッズなしモデルとの比較も行う。
6. 過剰に言ってはいけないこと:
   - Benter方式なら勝てるとは言わない。
   - 現代の日本競馬にそのまま通用すると断定しない。
   - オッズを使えば必ず回収率が上がるとは言わない。
7. 実データで検証すべきこと:
   - 独自予測とオッズのズレが回収率に効くか。
   - オッズ帯別に期待値が安定するか。
   - 人気馬・穴馬で予測確率が校正されているか。
   - 南関東データでも市場評価との差が再現性を持つか。

## B. Machine learning for sports betting: should model selection be based on accuracy or calibration?

1. 資料名: Machine learning for sports betting: should model selection be based on accuracy or calibration?
2. 種類: working paper
3. 信頼度: 高
4. 重要な教訓:
   - スポーツベッティングではaccuracyよりcalibrationが重要になり得る。
   - 競馬AIでも、勝率20%と予測した馬が長期的に本当に20%前後勝つかを見る。
   - 的中率だけでモデルを選ばない。
   - logloss、Brier score、calibration curveを評価に入れる。
5. このスキルに入れる判断ルール:
   - モデル比較では的中率だけでなく、確率校正指標を確認する。
   - 期待値評価を行う前に、確率が校正されているかを見る。
   - 確率帯別に予測確率と実績頻度を比較する。
6. 過剰に言ってはいけないこと:
   - この論文はNBAを対象にしているため、競馬で同じ結果になるとは断定しない。
   - calibrationだけ見ればよいとは言わない。
   - ROIの改善を保証しない。
7. 実データで検証すべきこと:
   - calibrationが良いモデルほど競馬のROIが安定するか。
   - 的中率が高いモデルとcalibrationが良いモデルのどちらが回収率で強いか。
   - 南関東データで確率帯別の校正が崩れる条件があるか。

## C. Favorite-Longshot Bias

1. 資料名: Favorite-Longshot Bias
2. 種類: 論文・研究概念
3. 信頼度: 高
4. 重要な教訓:
   - 人気薄は過剰に買われやすい可能性がある。
   - 穴馬は「人気がないから妙味」ではなく「人気以上に走る根拠があるか」で判断する。
   - 人気馬も過小評価される場合がある。
   - オッズ帯別、人気帯別に回収率を確認する。
5. このスキルに入れる判断ルール:
   - 穴馬評価では、人気がない理由と巻き返し材料を同時に確認する。
   - 人気馬を嫌う場合も、過剰人気の根拠をデータで確認する。
   - モデル評価は人気帯別・オッズ帯別に分解する。
6. 過剰に言ってはいけないこと:
   - すべての穴馬が過剰人気とは言わない。
   - すべての人気馬が買いとは言わない。
   - 日本の南関東競馬で同じ強さのバイアスがあると断定しない。
7. 実データで検証すべきこと:
   - 南関東でオッズ帯別の回収率がどうなっているか。
   - 人気薄狙いが本当に期待値を持つか。
   - 人気馬の方が損失が小さい傾向があるか。
   - 競馬場別、距離別、馬場別にバイアスの強さが変わるか。

   - 馬、騎手、展開、馬場、相互作用を見る必要がある。

   - 映像、パドック、ゲーム理論は拡張候補として扱い、初期実装の必須条件にしない。
   - 相互作用特徴量を検証仮説として整理する。
6. 過剰に言ってはいけないこと:
   - 最初から深層学習や映像解析を使うべきとは言わない。
   - ゲーム理論を入れれば精度が上がるとは断定しない。
   - 実装優先度を上げすぎない。

   - まず表データでベースラインを作る。
   - 映像やパドック情報を入れる前に、既存特徴量でどこまで改善できるか見る。
   - 南関東データで馬×騎手、脚質×枠、馬場×脚質の相互作用が効くか。

## E. Learning to Rank系の競馬予測研究

1. 資料名: Learning to Rank系の競馬予測研究
2. 種類: 論文・技術ドキュメント
3. 信頼度: 高
4. 重要な教訓:
   - 競馬は二値分類だけでなく、レース内順位予測として扱える。
   - 3着以内予測、勝率予測、順位予測は目的が違う。
   - 三連複・三連単に近づくほどランキング評価が重要になる。
   - NDCG、MAP、MRRを評価候補にする。
5. このスキルに入れる判断ルール:
   - 目的が単勝・複勝・三連系のどれかで、目的変数と評価指標を変える。
   - レース内順位を扱う場合は、分類指標だけでなくランキング指標も確認する。
   - NDCG、MAP、MRRは回収率の代替ではなく、上位候補の並びを測る補助指標として扱う。
6. 過剰に言ってはいけないこと:
   - LTRが常に分類モデルより良いとは言わない。
   - NDCGが高いから回収率が高いとは言わない。
   - 論文の結果をそのまま南関東に適用できるとは言わない。
7. 実データで検証すべきこと:
   - 3着以内分類とランキングモデルのどちらがROIに効くか。
   - NDCGやMAPと回収率が相関するか。
   - 三連系の買い目に使える順位精度があるか。
   - 南関東の少頭数・多頭数でランキング指標の意味が変わるか。

## F. 日本競馬の途中オッズ・直前オッズ変動研究

1. 資料名: Are Final Market Prices Sufficient for Information Aggregation? Evidence from Last-Minute Dynamics in Parimutuel Betting
2. 種類: working paper
3. 信頼度: 中
4. 重要な教訓:
   - 最終オッズだけを静的に扱うと危険な場合がある。
   - オッズは取得時点を必ず記録する。
   - 直前のオッズ変動は市場情報として別特徴量になり得る。
   - 最終オッズを使う場合は、実際にその時点で買える前提か確認する。
5. このスキルに入れる判断ルール:
   - オッズ特徴量には取得時刻を持たせる。
   - 最終オッズを使った評価は、実運用で利用可能かを明示する。
   - 途中オッズ、直前変動、最終オッズは別の情報として扱う。
6. 過剰に言ってはいけないこと:
   - 直前オッズを見れば勝てるとは言わない。
   - 最終オッズだけでは必ず不十分とは断定しない。
   - リアルタイム運用をすぐ前提にしない。
7. 実データで検証すべきこと:
   - オッズ取得時点ごとの予測性能。
   - 最終オッズ使用時のリークリスク。
   - 直前オッズ変動とROIの関係。
   - JRAで見られた傾向が南関東でも成立するか。

## G. scikit-learn Common pitfalls / Data leakage

- 資料名: scikit-learn Common pitfalls - Data leakage
- 種類: 公式資料・技術ドキュメント
- 何の根拠として使うか:
  - 予測時点で利用できない情報を学習や前処理に使うと、過大評価された検証結果になり得る根拠。
  - 前処理や特徴量変換も、学習データでfitし、検証・テストにはtransformだけを適用する根拠。
- このスキルに入れる判断ルール:
  - 競馬AIでは、レース後に確定する着順、払戻、上がり、通過順、確定人気、未来成績集計を予測特徴量にしない。
  - 集計特徴量、target encoding、標準化、欠損補完は、評価対象期間より前のデータだけで作る。
  - 特徴量重要度が高い特徴量ほど、まず取得時点とリークを疑って確認する。
- 使ってよい結論:
  - 予測時点で使えない情報はリークとして扱う。
  - リークがある検証成績は、実運用性能の根拠にしない。
- 使ってはいけない結論:
  - リークを除けば必ずROIが上がるとは言わない。
  - 特徴量が競馬的に有用そうなら取得時点を確認しなくてよいとは言わない。
- 南関東データで検証すべきこと:
  - 馬体重、オッズ、馬場状態、騎手変更、発走時刻変更の発表・取得時点が予測時点と整合するか。
  - 過去成績集計が未来レースを含んでいないか。

## H. scikit-learn TimeSeriesSplit

- 資料名: scikit-learn TimeSeriesSplit
- 種類: 公式資料・技術ドキュメント
- 何の根拠として使うか:
  - 時間順を持つデータでは、未来で学習して過去を評価する分割を避ける根拠。
  - train / valid / testを時間順に分ける根拠。
- このスキルに入れる判断ルール:
  - 競馬AIではランダム分割を安易に使わず、開催日・開催回・年月で時間順に分割する。
  - 検証データはモデル選定、テストデータは最終確認に使い、同じ期間で何度も最適化しない。
  - 年別、月別、開催別に性能が崩れるかを見る。
- 使ってよい結論:
  - 時系列検証は競馬AIの標準確認項目にする。
  - ランダム分割で高成績でも、時間順テストで崩れるなら実力と断定しない。
- 使ってはいけない結論:
  - TimeSeriesSplitを使えば検証が完全になるとは言わない。
  - 時間順分割だけでオッズ取得時点や集計リークが解決すると言わない。
- 南関東データで検証すべきこと:
  - 開催替わり、馬場傾向、クラス編成、騎手・厩舎の移動で性能が崩れるか。
  - 大井、川崎、船橋、浦和の開催順をまたいでも性能が安定するか。

## M. CatBoost paper: categorical features / ordered boosting / target leakage

- 資料名: CatBoost: unbiased boosting with categorical features
- 種類: 論文・技術ドキュメント
- 何の根拠として使うか:
  - カテゴリ特徴量やtarget statisticsが強力である一方、target leakageやprediction shiftに注意が必要な根拠。
  - ordered boostingや順序を意識したカテゴリ処理の考え方を参照する根拠。
- このスキルに入れる判断ルール:
  - 騎手勝率、厩舎勝率、馬×条件、騎手×競馬場のようなtarget encoding・比率特徴量は、過去データだけで作る。

  - 母数が少ないカテゴリ比率は、スムージングや最低件数条件の候補にする。
- 使ってよい結論:
  - カテゴリ特徴量は競馬AIで重要候補になり得る。
  - target encoding系特徴量は時系列リークを重点確認する。
- 使ってはいけない結論:
  - カテゴリ特徴量を増やせば必ず良くなるとは言わない。

- 南関東データで検証すべきこと:
  - 騎手×競馬場、厩舎×距離、馬×馬場の比率特徴量が母数込みで有効か。
  - スムージング前後で時系列テストとROIが改善するか。

- 何の根拠として使うか:
  - 競馬AIを目的定義、特徴量設計、予測モデル、評価の順で考える実務手順の参考。

- 使ってよい結論:

## W. 目的別モデル戦略への出典対応

- Benter:
  - オッズと独自予測のズレ、期待値、回収率、資金管理を分ける根拠として使う。
  - 勝率予測、期待値判定、単勝ROI評価で参照する。
  - Benter方式なら勝てるとは言わない。
- Calibration論文:
  - 確率校正、logloss、Brier score、calibration curveを確認する根拠として使う。
  - 勝率予測、複勝率予測、3着以内予測、期待値評価で参照する。
  - calibrationだけでROIが保証されるとは言わない。
- Favorite-Longshot Bias:
  - オッズ帯・人気帯別評価、人気薄の過大評価リスクを見る根拠として使う。
  - 穴馬評価、期待値判定、単勝・複勝ROI評価で参照する。
  - 人気薄だけ狙えば妙味があるとは言わない。
- Learning to Rank研究:
  - レース内順位予測、NDCG、MAP、MRRを候補にする根拠として使う。
  - 三連系の前段階、上位候補の並び評価で参照する。
  - NDCGが高いからROIが高いとは言わない。
- LightGBM / CatBoost公式:
  - GBDT・カテゴリ特徴量・ランキングモデルを候補にする根拠として使う。
  - 表データ分類、確率予測、順位予測で参照する。
  - LightGBMやCatBoostなら勝てるとは言わない。
- scikit-learn:
  - リーク、時系列分割、確率校正の基本ルールとして使う。
  - すべての目的変数で、データリーク確認、時系列 train / valid / test、calibration確認に参照する。
  - scikit-learnの手法を使うだけで検証が十分とは言わない。
