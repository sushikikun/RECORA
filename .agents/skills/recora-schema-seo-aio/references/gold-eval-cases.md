# Gold Eval Cases

Research date: 2026-06-21

Purpose: define test cases that reveal whether `recora-schema-seo-aio` behaves like a Recora expert rather than a generic SEO/schema assistant.

## Eval Rules

- Use Pass / Warning / Fail.
- Fail if the output promises ranking, traffic, rich result, AI citation, AI recommendation, or AI visibility.
- Fail if missing evidence is not marked `NEEDS_VERIFICATION`.
- Fail if dashboard/private pages are recommended for public indexation without explicit approval.
- Warning if the recommendation is directionally right but lacks dependency, leading indicator, or validation method.
- Pass only when the output is page-specific, evidence-separated, and implementable.

## Minimum Passing Output Traits

- Identifies page type.
- Identifies evidence level.
- Separates SEO technical, AIO/GEO extractability, structured data, content/page structure, and third-party/source ecosystem.
- Marks missing data `NEEDS_VERIFICATION`.
- Uses the required seven-section output when producing audits.
- Includes safe caveats for schema, robots, sitemap, bot access, llms.txt, and structured data.
- Avoids manipulative SEO and keyword stuffing.
- Provides implementable page/section/internal-link/schema/source actions.
