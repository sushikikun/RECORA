# AIO / GEO Failure Modes

Research date: 2026-06-21

Purpose: identify common failure patterns in Recora AIO/GEO recommendations and define correct behavior.

| Failure mode | Description | Why it is dangerous | Detection signal | Correct behavior | Quality-gate trigger | Example bad output | Example corrected output |
|---|---|---|---|---|---|---|---|
| Schema万能視 | Treating schema as the main solution. | Encodes unsupported facts and overpromises impact. | Recommendation only says "add schema." | Add visible content first; schema mirrors it. | Schema/client-facing claim. | "Add schema to get cited." | "Add visible FAQ, then matching schema; citation is not guaranteed." |
| robots許可万能視 | Treating crawler access as AI exposure. | Confuses access with use. | "Allow bots, exposure improves." | Separate access, indexation, retrieval, citation. | Bot/crawler claim. | "Allow GPTBot to improve AI visibility." | "Check access; AI use remains unverified." |
| sitemap送信万能視 | Treating sitemap as citation/index guarantee. | Misleads client about discovery support. | "Submit sitemap, citations improve." | Use sitemap for discovery support only. | Sitemap guarantee. | "Sitemap will increase citations." | "Sitemap supports discovery, not guaranteed crawl/index/citation." |
| Branded prompt混入 | Mixing branded prompts into visibility/ranking. | Inflates visibility and ranking metrics. | Brand name in prompt set used for visibility. | Use branded for sentiment/recognition/misinformation; non-branded for discovery. | Measurement report. | "Visibility is high because branded prompts mention us." | "Branded prompts are excluded from visibility rate." |
| mention/citation混同 | Treating brand mention as source citation. | Overstates source support. | Answer names brand but cites another source or none. | Report mention and citation separately. | Client report metric. | "We were cited." | "We were mentioned but not cited." |
| recommendation/ranking混同 | Treating answer order as recommendation. | Overstates user guidance. | Listed high without endorsement. | Separate order/ranking from recommendation text. | Competitive report. | "Ranked #1, recommended." | "Listed first; recommendation language not present." |
| Third-party evidence不足 | Owned claims lack independent support. | AI answers may rely on external sources. | Competitors cited from directories/media; client has none. | Split owned-page and third-party actions. | Source gap action. | "Rewrite our page to solve it." | "Owned page clarifies facts; independent proof remains needed." |
| Source gap自社ページ万能視 | Assuming all gaps are owned-page issues. | Misses external ecosystem work. | Gap source type is third-party. | Classify resolution type. | Source gap. | "Add a page and gap solved." | "Create page plus pursue legitimate third-party profile correction." |
| One-shot AI observation | Drawing conclusions from one answer. | AI answers vary. | Single prompt/provider/date. | Label sample-bound and remeasure. | Client claim. | "AI always says..." | "In this measured answer..." |
| Screenshot-only overclaim | Judging crawl/index/schema from screenshot. | Invents technical facts. | Only screenshot provided. | Use lightweight audit and mark technical data `NEEDS_VERIFICATION`. | Audit output. | "Schema is invalid." | "Schema validity not inspected." |
| Competitor未検証断定 | Making competitor claims without sources. | Legal/trust risk. | No competitor source evidence. | Mark `NEEDS_VERIFICATION`; request sources. | Comparison page. | "Competitor lacks this feature." | "Competitor feature claim needs verification." |
| fake AggregateRating | Adding ratings without real reviews. | Misleading structured data. | No visible review source/count. | Reject Review/AggregateRating. | Schema proposal. | "Add 5-star rating." | "Do not propose rating schema without real visible reviews." |
| keyword stuffing | Repeating keywords to influence systems. | Spam risk, poor UX. | Unnatural repetitions. | Add useful sections and entity clarity. | Content recommendation. | "Repeat AI SEO diagnosis 20 times." | "Answer buyer questions with natural terminology." |
| manipulative SEO | Fake reviews, fake citations, doorway pages, hidden text. | Trust and policy risk. | Deceptive tactic. | Suppress and route to quality gate. | Any manipulative tactic. | "Create fake directory listings." | "Pursue legitimate profile completion or earned coverage." |
| hidden/gated reliance | Critical facts only in hidden UI/gated content. | Reduces user and crawler accessibility. | Key info only behind interaction/login/PDF. | Add visible summary and crawlable page text. | Public page audit. | "PDF has details, enough." | "Add visible HTML summary and link PDF as support." |
| PDF-only critical info | Important evidence only in PDF. | Extraction/crawl uncertainty. | No HTML equivalent. | Provide HTML summary with evidence labels. | Report/research page. | "Put all methodology in PDF." | "Publish methodology summary in HTML." |
| dashboard public indexation | Treating private dashboard as SEO target. | Privacy/security risk. | Dashboard route in page list. | Private by default; require approval. | Dashboard page. | "Add dashboard to sitemap." | "Do not expose/index without product/security approval." |
