# v0.5 Parser Citation Review Playbook

## CONFIRMED_FROM_FILES

- The existing skill already has parser, citation, source-to-claim, and testing references.
- The parser/citation snapshot provides concrete scripts and outputs that show current parsing, normalization, and candidate generation surfaces.

## USER_PROVIDED_CONTEXT

- v0.5 must strengthen the skill without editing the existing skill directly.

## NEEDS_VERIFICATION

- Any future repository snapshot must be inspected again before applying this playbook.
- Parser and citation extraction implementation not fully inspected. Exact extraction behavior remains NEEDS_VERIFICATION.

## RECOMMENDED_ARCHITECTURE

Review sequence:

1. Inventory parser, provider, citation, source, and output artifacts.
2. Identify raw answer preservation and provider payload preservation.
3. Trace mention extraction from raw answer to mention rows and candidate evidence.
4. Check prompt echo, seed contamination, URL-only mention, alias collision, and Japanese normalization.
5. Trace citation extraction from provider payload to original URL, canonical URL, domain, occurrence count, raw citation, and cited span.
6. Check source-domain normalization and classification provenance.
7. Require source-to-claim status before citation-backed candidates.
8. Check parser and extraction versions and replay path.
9. Require golden fixtures for every identified bug class.
10. Produce file-impact maps and acceptance criteria only; do not edit app code in review mode.

## DO_NOT_ASSUME

- Do not invent parser behavior when files are absent.
- Do not decide quality-gate publication outcomes.
- Do not run external APIs, crawlers, browsers, database commands, or migrations in review-only calibration.
