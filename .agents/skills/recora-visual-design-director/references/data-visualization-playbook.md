# Data Visualization Playbook

Use this when proposing charts, graphs, diagrams, report visuals, dashboard UI, or card-level data shapes for Recora. Charts exist to clarify product understanding, not to fake proof.

## Universal Rules

- Treat numbers as sample data unless verified evidence is supplied.
- Do not make sample values look like measured customer results.
- Use charts to explain structure, comparison, status, or workflow.
- Do not use color alone to communicate meaning; pair color with label, icon, chip, pattern, or text.
- Keep mobile density readable at 375px.
- Avoid false precision. Prefer rounded sample values and clear labels.
- Put "Sample", "サンプル", "参考表示", or equivalent near the chart when data is illustrative.
- Keep Product Truth boundaries: no unsupported provider coverage, real-time monitoring, automatic collection, or improvement guarantees.

## Horizontal Bar Chart

Use for:

- visibility share sample.
- competitor comparison.
- source type distribution.

Rules:

- 3-5 bars max in LP cards.
- Show labels and values outside or beside bars.
- Use deep green for target/primary only when semantically appropriate.
- Use neutral colors for competitors unless highlighting a risk.

Avoid:

- tiny labels inside narrow bars.
- exact decimals.

## Stacked Bar

Use for:

- mention / recommendation / citation status mix.
- source categories.

Rules:

- Limit to 3 segments.
- Include a legend with text labels.
- Use subdued tones.
- On mobile, convert to segmented rows if needed.

## Ranking Bar

Use for:

- competitor ranking mini cards.
- query-level sample ranking.

Rules:

- Include rank number, label, bar, and status chip.
- Use sample label and method note.
- Avoid showing "winner" claims without evidence.

## Small Line Chart

Use for:

- abstract trend shape in a mock.
- activity over sample checks.

Rules:

- Use as texture, not proof.
- Do not imply continuous measurement unless Product Truth confirms.
- Label as sample trend or display example.

Avoid:

- axes that imply real time series.
- fine-grained plotted values.

## KPI Card

Use for:

- visibility score.
- query count.
- source count.
- recommendation count.

Rules:

- One large number, one label, one status/method note.
- Use sample label if illustrative.
- Keep numbers visually strong but not sensational.

## AI Answer Log Card

Use for:

- representing observed answer excerpts.
- showing mention vs recommendation vs citation.

Rules:

- 2-3 text lines max.
- Use neutral abstract UI, not provider-branded UI.
- Include brand/competitor tags and source chips.
- Mark sample/provider support carefully.

## Source Domain List

Use for:

- cited domains.
- referenced sources.
- source-to-claim status.

Rules:

- Show domain, source type, status, and small bar or chip.
- Distinguish source presence from support.
- Use `unknown` or `not reviewed` when status is not verified.

## Competitor Comparison Table

Use for:

- sample report preview.
- dashboard mock.

Rules:

- 3-5 columns max.
- Use rows for criteria: mention, recommendation, source, category fit, proof page.
- On mobile, convert to stacked cards or key rows.

## Persona Heatmap

Use for:

- Gumshoe-like buyer/persona analysis without copying its UI.

Rules:

- Rows: persona or buying criteria.
- Columns: Recora target and competitors, or criteria categories.
- Use 3 levels only: low / medium / high.
- Include text labels, not only color.
- Keep sample label visible.

## Before/After Improvement Diagram

Use for:

- showing report-to-action concept.
- explaining page improvement direction.

Rules:

- Before: unclear source, missing proof, weak category association.
- After: clearer page, structured proof, source-ready explanation.
- Present as concept, not guaranteed outcome.

## 3-Step Flow Diagram

Use for:

- diagnose -> compare -> improve.
- measure -> inspect sources -> create actions.

Rules:

- 3 steps max for LP.
- Each step has icon, label, one-line explanation, and mini UI cue.
- Do not imply automated execution unless Product Truth confirms.

## Report Paper Diagram

Use for:

- sample report preview.
- final proof section.

Rules:

- Show paper-like page, score cards, small table, method note, and sample label.
- Make it export/share-safe.
- Do not use fake client logos or outcome metrics.

## Avoid These Charts

- Meaningless pie charts.
- Wavy decorative lines.
- 3D graphs.
- Radar charts.
- Charts with precision that looks like real measurement when data is sample.
- Charts with labels too small to read.
- Charts used only as decoration without a decision they support.
