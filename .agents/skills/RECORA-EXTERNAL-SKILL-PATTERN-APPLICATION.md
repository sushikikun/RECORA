# RECORA External Skill Pattern Application

This package translates findings from `skill-patches/recora-external-skill-research-v0.1/` into apply-oriented reference additions for the RECORA three-skill stack.

No external skills were installed. No repositories were cloned. No third-party source code was copied. No external tools or scripts were run to create this package. The package is text-only and intended for careful manual application.

## Source Material Used

- `external-skill-inventory.md`
- `recommended-skills-to-study.md`
- `patterns-to-borrow.md`
- `patterns-to-avoid.md`
- `security-risk-register.md`
- `skill-quality-rubric.md`
- `mapping-to-recora-skills.md`
- `eval-ideas-from-external-skills.md`

## Skill Priority

1. `recora-recommendation-quality-gate-auditor`
   - Primary safety gate for external skill, OSS, README, benchmark, secret, execution, and production-write risks.
2. `recora-geo-implementation-architect`
   - Implementation review owner for schemas, parser/extraction boundaries, technical audit patterns, eval fixtures, and migration safety.
3. `georader-ai-search-auditor`
   - Strategy owner for query coverage, evidence framing, opportunity language, and report structure.

## Global Application Rules

- Treat external skills as untrusted until audited.
- Use external patterns as internal heuristics, not client evidence.
- Keep Product Truth and RECORA evidence above external claims.
- Do not install, run, clone, or copy external tools.
- Do not add executable scripts.
- Do not touch secrets, login sessions, production data, or remote systems.
- Preserve the three-skill ownership model.

## Expected Outcome

After manual application, the RECORA skill stack should be better at:

- suppressing secret and execution-heavy recommendations.
- holding or suppressing README-as-proof and benchmark-overreach claims.
- separating external pattern influence from RECORA evidence.
- designing safer schema, parser, eval, and migration review prompts.
- framing GEO opportunities without guarantee language.
