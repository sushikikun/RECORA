# SEO / GEO Expert Framework

Use this reference to run broad specialist analysis. This is not an official GEO standard.

## GEORADER Expert Review Framework

Label this framework as `GEORADER_ASSUMPTION` unless the individual point is separately backed by a source.

| Area | What To Evaluate | Typical Evidence |
|---|---|---|
| Technical Discoverability | crawlability, indexability, rendering, mobile usability, sitemap/robots, canonical signals, structured data opportunity | inspected HTML, robots.txt, sitemap, crawl export, Search Console export, browser/runtime output |
| Search Intent Coverage | non-branded discovery, problem-aware, comparison, branded, pricing, industry, region, and conversion intents | keyword research, query set, SERP/AI answer samples, user-provided demand data |
| Topical Authority | hub/spoke depth, glossary, FAQ, comparison, industry/use-case pages, internal links, content uniqueness | site architecture, content inventory, internal link map |
| Entity / Brand Authority | company identity, author expertise, external mentions, reviews, directory/profile consistency, proof assets | about page, author pages, third-party profiles, press, case studies |
| AI Citation Readiness | claim-source alignment, citable facts, cited URLs/domains, source diversity, extractable blocks | page text, citations, AI answers, report data |
| Competitive SERP / AI Gap | competitors appearing in SERPs or AI answers where the brand is absent or weak | SERP evidence, AI answer outputs, competitor pages |
| Conversion SEO | CTA clarity, proof, objection handling, form trust, report clarity, low-friction next steps | page/report copy, funnel screenshots, forms, analytics if supplied |
| Report / Product Value | usefulness of GEORADER diagnosis, client actionability, severity quality, sales readiness, monthly roadmap | report template, sample report, product docs, roadmap |

## Review Sequence

1. State evidence labels before making strategic claims.
2. Identify whether the task is product strategy, client strategy, report QA, or a narrow review.
3. Score or prioritize only where evidence supports it.
4. Convert findings into P0/P1/P2.
5. Translate technical findings into client-facing business impact.
6. Keep implementation as a plan only.

## Scoring Guidance

- Use 0-5 or 0-100 only as GEORADER's internal review scale.
- Explain what raised or lowered the score.
- Do not imply that the score is used by Google, OpenAI, Gemini, Perplexity, or any AI platform.
- Poor scores should be visible; they help clients understand what to fix.
