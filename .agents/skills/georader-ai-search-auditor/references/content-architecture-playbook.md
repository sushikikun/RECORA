# Content Architecture Playbook

Use this reference when turning GEO/SEO findings into page and content recommendations.

## Architecture Principles

- Build for search intent and buyer decision support, not page count.
- Use hub/spoke structure for major topics.
- Use internal links to connect education, comparison, proof, and conversion.
- Avoid doorway pages, thin template pages, and keyword-swapped clones.
- Make claims easy to cite with clear page-level evidence.

## Page Types

| Page Type | Purpose | Typical Evidence Label |
|---|---|---|
| Topic hub | Define a major topic and route users to subtopics | `INDUSTRY_PRACTICE` |
| Glossary | Explain terms and reduce entity ambiguity | `INDUSTRY_PRACTICE` |
| FAQ | Answer natural-language questions and objections | `INDUSTRY_PRACTICE` |
| Comparison | Help AI/users compare alternatives fairly | `INDUSTRY_PRACTICE` |
| Industry page | Map service to vertical-specific problems | `INDUSTRY_PRACTICE` or `GEORADER_ASSUMPTION` |
| Use-case page | Connect problem, workflow, proof, and CTA | `INDUSTRY_PRACTICE` |
| Case study | Provide proof and measurable outcomes | `CONFIRMED_FACT` when real |
| Methodology | Explain how diagnosis/scoring/reporting works | `GEORADER_ASSUMPTION` unless based on inspected implementation |
| Sample report | Show report value while labeling sample data | `GEORADER_ASSUMPTION` plus `CONFIRMED_FACT` for inspected artifact |

## GEORADER Priority Content

Prioritize:

- GEOとは.
- AI検索最適化とは.
- SEOとGEOの違い.
- ChatGPTで自社名が出ない理由.
- AI検索で競合だけが出る理由.
- GEO診断レポートの見方.
- 業界別AI検索対策.
- ChatGPT / Gemini / Perplexity 別の見え方.
- 改善前後の事例.
- 代理店向け販売資料.
