# Specialist Modes

Use this reference to select the mode and avoid mixing audit, strategy, report, and implementation work.

## Mode Selection

Choose one primary mode. Add supporting modes only when needed.

| Mode | Use When | Main Output |
|---|---|---|
| `georader-specialist-strategy` | GEORADER本体, diagnosis logic, CLI/report QA, paid diagnosis, monthly roadmap, product/report value | Internal evidence-labeled strategy and P0/P1/P2 roadmap |
| `client-geo-seo-ai-search-strategy` | Client brand/domain/category asks for GEO, SEO, AI visibility, citation readiness, or organic discovery strategy | Client-ready diagnosis plus internal improvement notes |
| `technical-discoverability-review` | Crawlability, indexability, rendering, structured data, agent accessibility, or technical SEO affects AI/source visibility | Technical findings with evidence labels |
| `search-intent-map` | User asks for query demand, intent map, non-branded discovery, or diagnosis prompts | Intent map and query taxonomy |
| `topical-authority-plan` | User asks what content/pages to build or how to cover a topic | Hub/spoke and content architecture plan |
| `entity-brand-authority-review` | User asks why the brand/entity is not understood, cited, trusted, or recommended | Entity clarity and proof-asset recommendations |
| `ai-citation-strategy` | User supplies AI answers/citations or asks how to become citable | AI answer analysis and citation-readiness plan |
| `competitor-serp-ai-gap` | User asks who wins in SERP/AI answers or why competitors are recommended | Competitor gap matrix and action plan |
| `conversion-seo-review` | User asks how search/AI traffic converts into diagnosis, consultation, report, or monthly service | Conversion SEO and trust recommendations |
| `report-quality-review` | User gives a report, sample PDF text, report JSON, or report template | Report QA findings and rewritten client-facing sections |
| `trust-risk-audit` | User mentions forms, provisional backend, delivery, payment, email, "looks submitted", or trust risk | P0 trust-risk list and safer wording |
| `implementation-plan-only` | User asks what to build next or asks to implement while this skill is active | P0/P1/P2 implementation plan without file edits |

## Mode-Specific Minimums

For `georader-specialist-strategy`, include:

- current truth vs inspected behavior.
- evidence labels.
- diagnosis/report/product value.
- trust risk.
- paid diagnosis or monthly roadmap implications.

For `client-geo-seo-ai-search-strategy`, include:

- target brand/domain/category/region.
- AI visibility and competitor gap.
- technical discoverability.
- intent coverage.
- topical/entity authority.
- citation readiness.
- conversion SEO.
- client-facing summary.

For `technical-discoverability-review`, include:

- crawlability and indexability evidence or not-inspected note.
- rendering and structured data detection limits.
- agent accessibility opportunities.
- no overclaims without official/current evidence.

For `search-intent-map`, include:

- intent buckets.
- query examples.
- current coverage if known.
- recommended assets.
- evidence label for demand.

For `report-quality-review`, include:

- evidence quality.
- severity quality.
- client-readiness.
- unsupported or falsely official claims.
- rewrite candidates.

For `trust-risk-audit`, include:

- exact implied claim.
- evidence missing.
- user harm.
- safer UI/report language.
