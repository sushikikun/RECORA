# External Eval Patterns For Query Coverage

Purpose: borrow eval-like coverage thinking for GEO / SEO / AIO / LLMO strategy without turning eval scores into client proof.

## Core Rule

External eval patterns can improve query set design, coverage review, and evidence ledgers. They cannot prove client AI visibility, citations, rankings, traffic, or revenue.

## Coverage Dimensions

| Dimension | Review use |
|---|---|
| Prompt set coverage | Ensure RECORA considers branded, non-branded, comparison, problem-aware, feature, industry, local, and objection queries. |
| Persona coverage | Separate executive, practitioner, buyer, evaluator, and technical-user intents when relevant. |
| Buyer-stage coverage | Map awareness, consideration, comparison, purchase, retention, and objection-stage queries. |
| Provider/mode coverage | Record which AI/search provider or answer mode was measured, if any. |
| Citation opportunity coverage | Identify where client-owned or third-party sources could support answer claims. |
| Competitor gap coverage | Track observed competitor mentions/citations without overstating durable market position. |

## Evidence Requirements

| Claim type | Required evidence before client-facing use |
|---|---|
| Current AI visibility | Fresh RECORA measurement or supplied answer export. |
| Citation gap | Source/citation observation and source-to-claim status. |
| Competitor gap | Query family, competitor set, observed answer evidence, and seed-risk review. |
| Query opportunity | Intent rationale and clear `NEEDS_VERIFICATION` if not measured. |
| Coverage score | Internal RECORA assumption only; not an official or external score. |

## Safe Output Pattern

Use:

"This query family is underrepresented in the current evidence set and should be measured before making a client-facing visibility claim."

Avoid:

"The eval coverage score proves the client has low AI visibility."
