# Recora Current Truth

Use this reference when a task mentions Recora, future Recora adoption, Recora client reports, Recora positioning, or migration from GEORADER language to Recora language.

## Confirmed For This Skill

- Recora is a product-name-ready evolution path for GEORADER.
- This skill may help produce evidence-scoped GEO / SEO / AI search strategy, source-grounded reports, and planning language for Recora adoption.
- This skill must not implement Recora, edit Recora product files, create Recora backend behavior, modify Recora LP/app/CLI/database/production systems, or touch secrets.

## Default Evidence Labels

- Use `GEORADER_ASSUMPTION` for Recora strategy, packaging, report structure, scoring, roadmap, positioning, and naming choices unless stronger evidence is supplied.
- Use `NEEDS_VERIFICATION` for Recora business claims, product capabilities, current customers, pricing, revenue, adoption timeline, platform support, automation, crawler measurements, and AI citation outcomes.
- Use `CONFIRMED_FACT` only for Recora facts supplied by the user or verified from inspected artifacts in the current task.
- Use `OFFICIAL_SOURCE` only for platform guidance from official docs, not for Recora's internal framework.
- Use `RESEARCH_BACKED` only for named research; do not use it for Recora claims unless the research directly supports the claim.

## Allowed Recora Language

- "Recora can be treated as a future-facing brand wrapper for GEORADER's evidence-based AI search audit and improvement workflow."
- "Recora's report structure should separate verified facts, official-source guidance, research-backed findings, and assumptions."
- "Recora should avoid claiming guaranteed AI citation or official GEO-standard status."

## Disallowed Recora Language

- "Recora is an official GEO standard."
- "Recora guarantees ChatGPT, Gemini, Claude, Perplexity, or AI Overview citation."
- "Recora's crawler policy ensures AI systems will cite the site."
- "Recora has verified customers, revenue, pricing, platform integrations, automated diagnosis, or delivery workflows" unless current evidence proves it.
- "Recora has implemented app, LP, backend, database, CLI, payment, email, or production features" unless inspected evidence proves it.

## Client-Facing Rule

In client-facing Recora reports, use Recora as the report or service name only when the user asks for it or supplied materials already use it. If the facts are not verified, write:

- "This section is based on Recora's internal evidence-scope framework."
- "Current platform visibility and citation status require fresh query testing."
- "Implementation and measurement status were not inspected for this report."

## Internal Rule

If Recora and GEORADER conflict, report the conflict and preserve the evidence hierarchy:

1. Inspected files or supplied artifacts.
2. User-provided facts.
3. Official platform sources.
4. Research sources.
5. This reference.
6. Strategic assumptions.
