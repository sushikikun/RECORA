# External GEO / SEO Patterns Adoption Policy

Use this policy when GEORADER strategy, audits, query maps, or report structures reference external SEO, GEO, LLMO, AIO, eval, or tool patterns.

## Core Rule

External SEO/GEO skills and tools can inspire query taxonomy, report sections, audit rubrics, evidence ledgers, and opportunity framing. They do not become RECORA evidence by themselves.

## Allowed Uses

- query taxonomy inspiration
- prompt set coverage checks
- buyer-stage and persona coverage
- citation opportunity framing
- competitor gap framing
- report section structure
- evidence ledger formatting
- technical readiness categories

## Prohibited Uses

- treating external claims as RECORA evidence
- guaranteeing AI visibility, AI citations, rankings, traffic, conversion, or revenue
- calling a third-party GEO framework an official standard without primary-source support
- turning external tool outputs into final client proof without RECORA observations
- making final `auto_publish` decisions; that belongs to `recora-recommendation-quality-gate-auditor`

## Evidence Labels

- `EXTERNAL_GEO_PATTERN`: external GEO/SEO/eval pattern used as strategy structure.
- `NEEDS_VERIFICATION`: external claim that needs RECORA evidence or primary-source confirmation.
- `GEORADER_ASSUMPTION`: GEORADER internal strategy framing.
- `CONFIRMED_FACT`: user-provided or RECORA-measured evidence.

## Safe Language

- Safe: "Use this query taxonomy as a coverage checklist for RECORA measurement planning."
- Safe: "This is a citation opportunity, not proof that the AI answer cites the page."
- Unsafe: "This external tool proves the client will be cited."
- Unsafe: "This tactic guarantees rankings."
