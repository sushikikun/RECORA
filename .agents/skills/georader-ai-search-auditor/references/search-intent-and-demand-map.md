# Search Intent And Demand Map

Use this reference when creating client strategy, query sets, or content roadmaps.

## Intent Buckets

| Intent | Example Shape | GEORADER Use |
|---|---|---|
| Non-branded discovery | `{CATEGORY} おすすめ`, `{REGION} {SERVICE} 会社` | Test whether the brand appears without prompting |
| Problem-aware | `{TARGET} が {PROBLEM} で困ったとき` | Map pains to service pages and educational content |
| Comparison | `{BRAND} vs {COMPETITOR}`, `{CATEGORY} 比較` | Identify shortlist and competitor narrative gaps |
| Alternative | `{COMPETITOR} 代替`, `{CATEGORY} 他社` | Capture switcher and indirect competitor demand |
| Branded verification | `{BRAND} とは`, `{BRAND} 評判` | Check entity clarity and trust |
| Pricing / cost | `{CATEGORY} 費用`, `{SERVICE} 相場` | Improve conversion and AI answer specificity |
| Industry | `{INDUSTRY} 向け {CATEGORY}` | Build vertical authority |
| Region | `{REGION} の {CATEGORY}` | Support local or Japan-market demand |
| Evidence / citation | `{BRAND} 事例`, `{BRAND} 実績`, `{BRAND} 出典` | Make proof assets discoverable |
| Conversion | `{CATEGORY} 相談`, `{SERVICE} 導入支援` | Connect demand to CTA and report offer |

## Demand Handling

- If keyword volume or Search Console data is not supplied, mark demand estimates as `NEEDS_VERIFICATION`.
- If the user supplies actual query data, mark it `CONFIRMED_FACT`.
- Use query taxonomy for AI answer testing; use intent map for content and conversion strategy.
- Avoid creating pages only because a keyword exists. Tie each page to buyer value, evidence, and GEORADER report usefulness.

## GEORADER-Specific Notes

- Prioritize non-branded discovery and competitor comparison because GEORADER's core promise is detecting whether competitors are recommended instead.
- Include branded verification because entity confusion can weaken AI answers.
- Include citation/source verification because AI answers need evidence to support recommendations.
